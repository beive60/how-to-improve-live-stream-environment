# EP019 / CM-01: OBSプロファイルをGitで管理する - 設定のバージョニングと安全な復元

## 必要な動画素材

### 画面キャプチャ

- OBSプロファイルの保存ディレクトリ構造（%APPDATA%\obs-studio\ 以下のフォルダ・ファイル一覧）
- OBSのプロファイルメニュー画面（プロファイル > 管理 / シーンコレクション > 管理）
- ターミナルでの git init / git add / git commit コマンド実行画面
- VS CodeやGitHub Desktop等でのdiff表示画面（設定変更箇所のハイライト）
- .gitignore ファイルの編集画面（除外すべきファイルの記述）

### デモ映像

- OBSプロファイルディレクトリの構造を確認する操作デモ
- Gitリポジトリを初期化し、初回commitを作成する手順デモ
- OBSで設定を変更し、変更内容をgit diffで確認 → commitする一連の操作デモ
- 設定を壊してしまった想定で、git revert / git checkout で復元するデモ
- ブランチを使って実験的な設定変更を安全に試す操作デモ
- GitHub / GitLabへのリモートプッシュによるバックアップ手順デモ

### 図解・テロップ素材

- OBSプロファイルのディレクトリ構成図（各ファイルの役割を注釈付き）
- Gitワークフロー図（変更 → add → commit → push の流れ）
- .gitignoreに含めるべきファイルと管理すべきファイルの分類表
- バージョン管理のメリット一覧（変更追跡 / 復元 / バックアップ / 複数環境共有）

## 必要機材

### 所有済み

- PC1 / PC2（OBSプロファイルディレクトリの確認・ Git操作デモ用）

### 追加購入

なし。本エピソードはソフトウェア完結（Git, VS Code / GitHub Desktop）。
