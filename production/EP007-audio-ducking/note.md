# EP007 / AE-07: Audio Ducking / Sidechain Compression - ゲーム音をマイク音声に連動して自動減衰させる

## 必要な動画素材

### 画面キャプチャ

- OBS音声フィルタでのVSTコンプレッサー設定画面（サイドチェイン入力の指定）
- サイドチェイン対応VSTプラグインのUI画面（ReaComp / TDR Kotelnikov GE 等）
- OBSの複数音声トラック設定画面（マイク / ゲーム音 / BGM の分離）
- OBS詳細オーディオプロパティ画面

### デモ映像

- ダッキングなしの状態デモ（ゲーム音やBGMと声が同音量でぶつかり聞き取りにくい状態）
- ダッキング適用後のデモ（発話するとゲーム音が自然に下がり、黙ると元に戻る）
- Thresholdパラメータの効果デモ（反応する音量レベルの調整）
- Ratioパラメータの効果デモ（ゲーム音をどの程度下げるかの調整）
- Attack / Releaseパラメータの効果デモ（減衰の速さ・復帰の速さ）
- 過剰なダッキング設定の悪い例デモ（ゲーム音がポンピングする状態）

### 図解・テロップ素材

- サイドチェインコンプレッションの信号フロー図（メイン入力 / サイドチェイン入力 → コンプレッサー → 出力）
- ダッキングの動作タイムライン図（マイク音声レベル vs ゲーム音声レベルの時間推移）
- 推奨パラメータ設定値の目安表（ゲーム音用 / BGM用）

## 必要機材

### 所有済み

- マイク: Audio-Technica AT-2020

### 追加購入（必須）

- DSP内蔵オーディオIF（AE-05/AE-06で購入済みの想定）
  - サイドチェインコンプレッションの実装には複数の音声入力チャンネルの分離が必要
  - VT-4は単一チャンネル入力のため、マイク/ゲーム音/BGMの並列ルーティングに非対応
  - OBS内VSTのみで実装するデモであればVT-4でも可能だが、オーディオIFレベルのルーティングを併用するデモにはDSP内蔵IFが必要
