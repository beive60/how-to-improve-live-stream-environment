# EP016 / NE-03: Ingest Server経路最適化 - RTMPSサーバーへのTraceroute分析

## 必要な動画素材

### 画面キャプチャ

- コマンドプロンプトでの tracert コマンド実行結果画面（YouTube Ingestサーバー宛て）
- コマンドプロンプトでの tracert コマンド実行結果画面（Twitch Ingestサーバー宛て）
- OBSの配信設定画面（サーバー選択ドロップダウン）
- Twitch Inspector等の配信品質確認ツール画面
- WinMTR等のネットワーク経路品質計測ツール画面

### デモ映像

- 複数のIngestサーバーに対してtracerouteを実行し、ホップ数・レイテンシーを比較するデモ
- WinMTRで長時間計測し、パケットロスが発生しているホップを特定する操作デモ
- OBSでIngestサーバーを手動で切り替え、配信品質の変化を確認するデモ
- 最適なサーバーを選択した場合と自動選択の場合のビットレート安定性比較デモ
- ISPのルーティング変更（時間帯による経路変化）を示す計測デモ

### 図解・テロップ素材

- ネットワーク経路の可視化図（ユーザー → ISP → IX → CDN → Ingestサーバー のホップ構成）
- tracerouteの出力の読み方を説明する注釈付き図
- YouTube / Twitchの主要Ingestサーバーの所在地マップ
- 経路品質の判断基準表（RTT / パケットロス率の許容範囲）

## 必要機材

### 所有済み

- PC1 / PC2（tracert, WinMTR実行用）

### 追加購入

なし。本エピソードはソフトウェア完結（tracert, WinMTR, OBS配信設定）。
