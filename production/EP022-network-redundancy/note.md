# EP022 / CM-04: 回線冗長化戦略 - バックアップ接続の設計と自動切替

## 必要な動画素材

### 画面キャプチャ

- Windowsネットワーク接続画面（複数アダプタの状態表示: 有線LAN + モバイル回線等）
- ルーターの WAN設定画面（マルチWAN / フェイルオーバー設定がある場合）
- 自動切替スクリプト（PowerShell / Python）のコード表示画面
- タスクスケジューラの設定画面（監視スクリプトの定期実行設定）
- OBS配信設定画面（自動再接続設定）

### デモ映像

- メイン回線（有線LAN）の障害をシミュレートするデモ（ケーブル抜去 / アダプタ無効化）
- 手動切替の場合の操作デモ（障害検知から復旧までの所要時間計測）
- 自動切替スクリプトによる自動フェイルオーバーデモ（検知 → 切替 → 配信継続）
- OBSの自動再接続機能が動作する様子のデモ
- 切替時間の計測結果表示（手動 vs 自動の所要時間比較）
- 回線復旧後のフォールバック動作デモ（自動的にメイン回線に戻る）

### 図解・テロップ素材

- 回線冗長化の構成図（メイン回線 / バックアップ回線 / 監視・切替機構）
- フェイルオーバーのシーケンス図（障害検知 → 切替判断 → 経路変更 → 再接続）
- 冗長化レベル別の構成比較表（手動切替 / スクリプト自動切替 / ハードウェア冗長化）
- 監視スクリプトのロジックフローチャート
- コスト vs 可用性のトレードオフ図

## 必要機材

### 所有済み

- PC2（配信PC。フェイルオーバーデモ用）
- 光回線（メイン）
- KDDI テザリング対応回線（バックアップ）

### 追加購入

なし。KDDIテザリングでバックアップ回線を確保済み。メイン回線障害のシミュレート（ケーブル抜去/アダプタ無効化）と自動切替スクリプトのデモが実施可能。
