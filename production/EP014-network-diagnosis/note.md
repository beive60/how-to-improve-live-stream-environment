# EP014 / NE-01: 配信が止まる・カクつく原因の切り分けガイド - ネットワーク診断フロー

## 必要な動画素材

### 画面キャプチャ

- OBS統計パネル（フレームドロップ数: Rendering Lag / Encoding Lag / Network の各カウンター）
- OBSログファイルの該当箇所（エラー行の表示）
- タスクマネージャー（CPU / GPU / メモリ / ネットワーク使用率の同時表示）
- コマンドプロンプトでの ping / tracert 実行結果画面
- Speedtest等のブラウザベース速度テスト画面
- リソースモニター（ネットワークタブ > プロセス別の帯域使用状況）

### デモ映像

- PC負荷起因のフレームドロップデモ（CPU/GPUが100%に張り付き、映像がカクつく状態）
- ネットワーク起因のフレームドロップデモ（帯域不足やパケットロスで配信が止まる状態）
- OBSログの確認手順の操作デモ（ヘルプ > ログファイル > 直前のログを表示）
- 診断フローに沿って原因を切り分けていく一連の操作デモ
- 各対処法の適用前後のビフォーアフター

### 図解・テロップ素材

- 診断フローチャート（スタート → Rendering Lag? → はい: CPU/GPU問題 → GPU使用率確認 ... の分岐図）
- OBS統計パネルの各指標の意味を説明する注釈付きスクリーンショット
- 原因パターン別の対処法早見表（CPU負荷 / GPU負荷 / ネットワーク / ディスクI/O）
- フレームドロップの3種類（Rendering / Encoding / Network）の違いを示す図

## 必要機材

### 所有済み

- PC1 / PC2（OBSの統計パネル・タスクマネージャーデモ用）

### 追加購入

なし。本エピソードはソフトウェア完結（OBS統計パネル、タスクマネージャー、ping/tracert、Speedtest）。
