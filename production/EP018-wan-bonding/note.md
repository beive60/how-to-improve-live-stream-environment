# EP018 / NE-05: WANボンディング実践 - Speedify等による回線冗長化

## 必要な動画素材

### 画面キャプチャ

- Speedifyのメイン画面（接続中の回線一覧・ボンディングモード選択）
- Speedifyの設定画面（Priority / Redundant / Speed モードの選択）
- Speedifyの統計画面（各回線のスループット配分表示）
- Windowsネットワーク設定画面（複数アダプタの状態表示）
- モバイルテザリング設定画面（スマートフォン側）

### デモ映像

- 複数回線（光回線 + モバイルテザリング）を同時接続する手順デモ
- ボンディング動作時のスループット計測デモ（単一回線 vs ボンディング後の速度比較）
- メイン回線を意図的に切断し、フェイルオーバーで配信が継続する様子のデモ
- Redundantモードでの冗長伝送デモ（パケットロス耐性の向上）
- OBSの統計パネルでフレームドロップが発生していないことの確認画面
- 使用するハードウェア（USBテザリング用ケーブル、モバイルルーター等）の実写映像

### 図解・テロップ素材

- WANボンディングの仕組み図（複数回線 → ボンディングソフト → 仮想アダプタ → OBS）
- ボンディングモード（Speed / Redundant / Priority）の動作比較図
- WAN冗長化手法の比較表（ボンディング vs ロードバランシング vs フェイルオーバー専用）
- コスト計算表（Speedify契約料 + モバイル回線料 vs 配信途切れのリスク）

## 必要機材

### 所有済み

- PC2（配信PC。Speedify動作用）
- KDDI テザリング対応回線（バックアップ回線として使用）
- 光回線（メイン回線）

### 追加購入

なし。Speedifyはソフトウェアライセンス（月額課金）。KDDIテザリングでモバイル回線を確保済み。
