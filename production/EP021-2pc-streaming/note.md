# EP021 / CM-03: 2PC配信構成設計 - NDI/Capture Cardによる負荷分散とフェイルオーバー

## 必要な動画素材

### 画面キャプチャ

- OBS NDIプラグインの設定画面（送信側: NDI Output Settings）
- OBS NDIソースの設定画面（受信側: NDI Source の追加と選択）
- キャプチャカードの設定画面（OBS > ソース > 映像キャプチャデバイス）
- ゲーミングPC側のタスクマネージャー（負荷分散前後のCPU/GPU使用率）
- 配信PC側のタスクマネージャー（エンコード負荷の表示）
- ネットワークアダプタの設定画面（NDI専用NICのIP設定）

### デモ映像

- 2PC構成のハードウェア全体を映した実写映像（ゲーミングPC、配信PC、スイッチ、ケーブリング）
- NDI方式での映像・音声伝送デモ（ゲーミングPC → LAN → 配信PC → OBSプレビュー）
- キャプチャカード方式での映像伝送デモ（HDMI出力 → キャプチャカード → 配信PC）
- NDIとキャプチャカードの画質・遅延比較デモ
- 1PC構成と2PC構成のCPU/GPU負荷比較デモ（同一ゲーム・同一配信設定での実測）
- フェイルオーバー動作デモ（メイン配信PCがダウンした想定でバックアップに切り替え）
- キャプチャカード本体の実写映像（端子・接続方法）

### 図解・テロップ素材

- 2PC構成の全体構成図（映像・音声・ネットワークの信号フローを網羅）
- NDI方式 vs キャプチャカード方式の比較表（画質 / 遅延 / コスト / 柔軟性）
- フェイルオーバー設計図（正常時 / 障害時の切替フロー）
- 必要機材・コスト一覧表
- 負荷比較グラフ（1PC vs 2PC）

## 必要機材

### 所有済み

- PC1: AMD Ryzen 7 7800X3D / AMD Radeon RX 7900 XT（ゲーミングPC）
- PC2: AMD Ryzen 5 5600X / NVIDIA GeForce RTX 3060（配信PC。NVENCエンコード担当）
- キャプチャカード: AVerMedia V2（HDMIパスのデモに使用）
- ネットワークスイッチ: TP-Link 5ポート ギガビット（NDIトラフィック用）
- 2PC間 1Gbps接続: 確立済み
- LANケーブル: 既存接続で対応

### 追加購入

なし。全ての必要機材を所有済み。
- AVerMedia V2: HDMI入力対応、OBS映像キャプチャデバイスとして認識可能
- TP-Linkスイッチ: NDI 1080p60（約100-150Mbps）に対して1Gbpsで十分な帯域
- NDI方式 vs キャプチャカード方式の比較デモが両方実施可能
