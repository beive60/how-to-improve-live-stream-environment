# Channel Growth Strategy

投稿スケジュール、KPIモニタリング、リスク管理、アウトソーシング、機材購入計画の戦略書

## 1. 投稿スケジュールとマイルストーン

### 1.1. 投稿ペース

 隔週（月2本）の投稿を目標とする。

隔週投稿を選択する理由:

- YouTubeのアルゴリズムは一定の投稿頻度を好む
- 月1本では「定期的に投稿するチャンネル」としてのシグナルが弱い
- 週1本は品質を維持できないリスクがある

### 1.2. 投稿曜日と時間

| 項目 | 推奨値 | 理由 |
| --- | --- | --- |
| 投稿曜日 | 土曜日 | ターゲット層（個人配信者/プロシューマー）が設定作業に取り組む週末の開始にあわせる |
| 投稿時間 | 17:00-18:00 JST | 週末の活動開始時間。YouTube Studioのアナリティクスで最適化する |
| Premiere（プレミア公開） | 初期は使用しない | 登録者がほぼいない状態でPremiereを使うと、チャットが空になり逆効果。登録者500人以上を目安に検討する |

### 1.3. マイルストーン

| 期間 | マイルストーン | 定量目標 | 戦略的意味 |
| --- | --- | --- | --- |
| Week 0-6 | チャンネル開設 + 第1本公開 | - | チャンネル名、アート、概要欄を技術チャンネルとして設計。最高品質の1本目で方向性を確立する |
| Month 1-3 | Phase 1前半（4-5本公開） | 動画平均CTR 5%以上、平均視聴維持率 50%以上 | アルゴリズムに「VTuber配信環境の技術チャンネル」と学習させる |
| Month 4-6 | Phase 1完了（9本公開） | Browse Featuresからの流入10%以上、登録者500人以上 | 検索以外のトラフィックソースが育ち始める。Community Tab解放 |
| Month 7-9 | Phase 2前半 | 月間視聴時間の前月比成長率10%以上、登録者1,000人以上 | 体系的コンテンツによるセッション時間の延伸。YPP資格到達 |
| Month 10-12 | Phase 2後半 | 登録者3,000人以上、Twitch同時接続10人以上 | クロスプラットフォーム導線の効果が現れる |
| Month 13以降 | Phase 3 + 展開 | コンサルティング/案件の実績 | 「人の価値」が経済的価値に転換される |

## 2. KPIとモニタリング

### 2.1. Primary KPI（最重要指標）

| KPI | 目標 | 計測方法 | 判断基準 |
| --- | --- | --- | --- |
| 平均視聴維持率 | 50%以上 | YouTube Studio > Analytics > Audience Retention | 50%未満が続く場合、動画構成を見直す |
| CTR（クリック率） | 5-10% | YouTube Studio > Analytics > Impressions | 3%未満が続く場合、サムネイル/タイトルを見直す |
| 登録者獲得率 | 動画視聴者の2-5%が登録 | YouTube Studio > Analytics > Subscribers | 新規チャンネルの成長速度の指標。1%未満が続く場合、チャンネルの訴求力を見直す |

### 2.2. Secondary KPI（補助指標）

| KPI | 目標 | 意味 |
| --- | --- | --- |
| Browse Features比率 | 全体の10%以上（Month 4以降） | アルゴリズムによるレコメンデーションが機能しているかの指標 |
| 平均セッション時間 | 2本以上の連続視聴 | プレイリストと導線設計が機能しているか |
| コメント率 | 視聴者の1%以上 | エンゲージメントの深さ |
| Twitch同時接続 | Month 10以降で10人以上 | クロスプラットフォーム導線の効果 |
| LTV貢献度（コンサルティング流入） | Month 13以降で技術相談の実績 | Phase 3コンテンツ（CM-03、DO-04、AE-08等）が高単価の技術相談へ直結しているかの指標。広告収益のROIが低いPhase 3コンテンツを正当に評価するための補完指標 |


### 2.3. 動画単位のPDCAサイクル

各動画の公開後、以下のタイミングでパフォーマンスを評価し、次の動画に反映する。

| タイミング | チェック項目 | アクション |
| --- | --- | --- |
| 公開後48時間 | CTR、初期視聴回数、コメント数 | CTRが3%未満ならサムネイル差し替えを検討する |
| 公開後1週間 | 視聴維持率カーブの形状、離脱ポイントの特定 | 次の動画の構成に離脱要因の排除を反映する。冒頭15秒以内に計測データ等の客観的証拠を提示する構成になっているかを確認し、推測ベースの解説動画との差別化が即座に確立されているかを評価する |
| 公開後1ヶ月 | トラフィックソースの比率、検索キーワード | SEO最適化の効果を確認し、説明文を微調整する |

## 3. リスク管理（成長戦略固有）

### 3.1. 初動失敗リスク

**リスク**: 最初の3本のCTR/維持率が低く、アルゴリズムがチャンネルを「低品質」と判定する

**対策**:

- 第1本は最もスコアが高いAE-01（OBS音ズレ）を使用し、制作に最大限の時間を投下する
- 公開前にサムネイルのA/Bテストを実施する（X/Discordでのアンケート）
- 外部流入（SNS、コミュニティ）で初動48時間のシグナルを強化する

### 3.2. 新規チャンネルの初期成長リスク

**リスク**: 登録者0人からの開始であるため、Browse FeaturesやSuggested Videosからの配信が得られるまでに時間がかかる。初期のインプレッション母数が限定的で、成長が遅い可能性がある。

**対策**:

- 初期のトラフィックソースはSearch（検索）とExternal（外部流入）を主軸とする。アルゴリズム依存のBrowse/Suggestedは初期には期待しない
- 動画公開と同時にSNS（X/Twitter）、技術コミュニティ（Reddit、Discord）で共有し、外部から視聴データを注入する
- 検索流入を最大化するため、明確な検索需要があるトピック（AE-01: OBS音ズレ等）を初期に配置する
- 外部流入で一定の視聴実績（視聴回数、維持率、セッション時間）が蓄積された段階で、アルゴリズムがBrowse/Suggestedへの配信を開始する
- 新規チャンネルにはデッドチャンネルの負債がないため、品質の高い動画が正当にアルゴリズムに評価される環境にある。焦らず品質を優先する

### 3.3. 燃え尽きリスク

**リスク**: 隔週投稿のペースを維持できず、長期間の投稿停止が発生する

**対策**:

- 常に2本分のストック（完成済み未公開動画）を持つことを目標にする
- 制作が遅延した場合、コミュニティ投稿やShorts（60秒以内の技術ティップス）で投稿頻度のシグナルを維持する
- 無理に隔週を維持するために品質を下げるくらいなら、月1本に落としても品質を維持する


## 4. アウトソーシング戦略

### 4.1. 基本原則: 「人の価値」に直結する作業は内製、それ以外は外注する

外注の判断基準は以下の通り。

- **内製必須**: 視聴者が「この人の価値」と認識する要素。技術的知見、語り、判断、パーソナリティに関わる作業
- **外注推奨**: 制作品質に寄与するが、「誰がやったか」が視聴者に見えない作業。定型化可能で、指示書があれば再現性がある作業

### 4.2. 内製/外注 分類マトリクス

| 作業 | 判定 | 理由 |
| --- | --- | --- |
| 技術調査・検証 | 内製必須 | 知見そのものがコンテンツの価値。外注不可 |
| 台本・構成作成 | 内製必須 | 語りの設計はパーソナリティに直結する |
| ナレーション収録 | 内製必須 | 声 = ブランド。TTS不使用の方針と同根 |
| RVCモデル管理・推論 | 内製必須 | 音声パイプラインの根幹 |
| 技術デモ画面収録 | 内製必須 | 操作の流暢さが技術的権威（Layer 2）を構成する |
| コミュニティ対応 | 内製必須 | コメント返信、X投稿は人格的信頼（Layer 3）の構成要素 |
| 戦略・分析判断 | 内製必須 | KPI評価とPDCAはチャンネルオーナーの責務 |
| サムネイル制作 | **外注** | テンプレート化により指示書ベースで発注可能 |
| 動画編集 | **外注** | 技術デモ収録と台本があれば、組み立ては外注可能 |
| テロップ・字幕制作 | **外注** | 台本から機械的に生成可能。精度チェックのみ内製 |
| チャンネルアート・ロゴ | **外注** | ブランド設計の実装。方向性の指示は内製 |
| BGM・SE選定 | **外注可** | ロイヤリティフリー素材のキュレーションを委託可。最終選定は内製 |
| 音声後処理（ラウドネス/EQ） | **自動化** | スクリプトで自動化すべき。外注の必要すらない |
| 英語字幕 | **外注** | 国際展開時に検討。Phase 2以降 |

### 4.3. 外注推奨タスクの詳細

#### 4.3.1. 動画編集

最も時間を消費する工程であり、外注効果が最も高い。
ただし、技術解説動画の編集には正確性が求められるため、段階的に移行する。

**外注範囲の段階的拡大**:

| 段階 | 外注範囲 | 内製で残す部分 | 移行条件 |
| --- | --- | --- | --- |
| Phase A | テロップ挿入、SE配置、BGM調整 | カット編集、技術デモの配置 | 編集テンプレート（プリセット）の確立後 |
| Phase B | カット編集を含む粗編集（ラフカット） | 技術的正確性の最終チェック、微調整 | 指示書フォーマットが安定した後 |
| Phase C | 完パケ納品（フルアウトソース） | 最終チェックのみ | 信頼できる編集者との関係構築後 |

**発注に必要な素材セット**:

- 台本（テロップ文言、タイミング指示を含む）
- ナレーション音声（RVC処理済み・後処理済み）
- 技術デモ画面収録（OBS等でキャプチャしたもの）
- VTuberアバター画像（State別PNGセット）
- SDイラスト素材
- 編集テンプレートファイル（トランジション、テロップスタイル、カラー設定のプリセット）

**コスト目安**:

| 外注段階 | 1本あたり | 月2本 x 年間 | 備考 |
| --- | --- | --- | --- |
| Phase A（テロップ/SE） | 5,000-10,000円 | 120,000-240,000円/年 | 指示書が明確であれば品質安定 |
| Phase B（ラフカット） | 15,000-30,000円 | 360,000-720,000円/年 | 編集者の技術理解度に依存 |
| Phase C（完パケ） | 30,000-50,000円 | 720,000-1,200,000円/年 | 収益化後に検討 |

**ROI**: 10-15分動画の編集に自作で8-15時間かかる場合、Phase Bで年間192-360時間を節約できる。月2本の隔週投稿ペースの持続可能性が大幅に向上する

**推奨**: Phase Aから開始し、最初の3-5本で編集テンプレートとワークフローを確立する。Phase Bへの移行はMonth 4以降を目安とする

### 4.4. 外注スケジュール

外注の開始時期をプロジェクトの進行に合わせて段階的に設計する。

| 時期 | 外注開始タスク | 理由 |
| --- | --- | --- |
| Month 1-3（Phase 1前半） | サムネイル制作 | 2-3枚自作してテンプレートを検証した後に移行 |
| Month 4-6（Phase 1後半） | 動画編集 Phase A | 編集テンプレートが安定した後に移行 |
| Month 7以降（Phase 2） | 動画編集 Phase B | 編集者との信頼関係が構築された後に移行 |

### 4.5. 外注における品質管理

**外注先の選定基準**:

- ポートフォリオで技術系コンテンツの制作実績を確認する
- テスト発注（1件）で品質とコミュニケーションコストを評価する
- 納品物のリビジョン（修正回数）を事前に合意する（2回まで等）
- 可能であれば固定の外注先と継続契約を結ぶ（品質安定、単価交渉、学習コスト削減）

**品質チェックフロー**:

1. 外注先に指示書と素材を納品する
2. 初稿を受領し、技術的正確性とブランドガイドラインとの適合性を確認する
3. 修正指示を出す（必要な場合）
4. 最終稿を承認する

#### 5.2.1. Beiveモデル

- VTS入力パラメータ: 10種（20マッピング）
- カバー対象: 顔回転/位置、目の開閉（L/R）、視線（L眼）、眉（L/R）、口の開閉、口の笑顔
- ARKit 52種のうち統合入力でカバー: 約20種
- 個別にデモ可能なBlend Shapes: 0種（全て統合入力に集約）
- 完全に欠落: 約32種（eyeSquint、jaw系3種、口周り21種、頬3種、鼻2種、tongueOut）

#### 5.2.2. twintaleモデル

- VTS入力パラメータ: 9種（12マッピング）
- カバー対象: 顔回転、目の開閉（L/R）、視線（R眼）、眉（統合）、口の開閉、口の笑顔、舌
- ARKit 52種のうち統合入力でカバー: 約18種
- 個別にデモ可能なBlend Shapes: 1種（tongueOut）
- 完全に欠落: 約34種

#### 5.2.3. エピソード別の判定

- **VT-03（ARKit vs Webcam比較）**: 両モデルとも対応可能。ジッター計測は生パラメータ値の数値比較であり、モデルのマッピング網羅性は計測精度に影響しない。映像デモにはBeiveを推奨（マッピング数が多く、眉の左右独立や身体の揺れも表現可能）
- **VT-04（Blend Shapes 52種解説）**: 両モデルとも不十分。52種の個別デモには、各Blend Shapeに対応するLive2D側のリグが必要。Full ARKit対応モデルの追加購入を推奨

#### 5.3.4. VT-04向けLive2Dモデル（要検討）

VT-04（Blend Shapes 52種完全解説）の制作には、ARKit 52種のBlend Shapesが個別にマッピングされたLive2Dモデルが必要。
現有のBeive/twintaleモデルでは口周り23種中21種、頬3種、鼻2種が欠落しており、個別デモに対応できない。

No.2が最有力な選択肢であるが、モデル上の視覚デモは対応パラメータのみに限定されるため、解説内容を「ARKit 52種のうち、モデルでカバーされているパラメータを中心に解説する」とする必要がある。

| 1 | nizima等でFull ARKit対応モデルを購入 | 10,000-50,000円 | 「ARKit 52 Blend Shapes対応」を明示しているモデルを選べば確実 |
